# 冯江威(COSARO)个人设计作品集网站 - 二级页面开发计划

## 任务理解

用户希望为现有的单页作品集网站添加多个二级页面，包括：
1. 项目分类汇总页 - 按类别展示所有项目
2. 单项目详情页 - 展示单个项目的完整信息
3. 关于我详情页 - 更详细的个人介绍
4. 服务详情页 - 设计服务详细介绍
5. 博客/文章页 - 设计文章和教程

## 当前状态分析

- **现有架构**: 单页 HTML 网站 (index.html)
- **样式系统**: 内嵌 CSS，使用 CSS 变量定义设计系统
- **图片资源**: 63 张作品图片，按 4 个类别存放
- **技术栈**: 纯静态 HTML + CSS + JavaScript，无框架依赖

## 需要创建的文件

| 文件名 | 用途 | 依赖 |
|--------|------|------|
| `assets/css/common.css` | 公共样式提取 | index.html 中的样式 |
| `assets/js/common.js` | 公共脚本提取 | index.html 中的脚本 |
| `projects.html` | 项目分类汇总页 | common.css, common.js |
| `project-detail.html` | 单项目详情页 | common.css, common.js |
| `about.html` | 关于我详情页 | common.css, common.js |
| `services.html` | 服务详情页 | common.css, common.js |
| `blog.html` | 博客列表页 | common.css, common.js |
| `blog-post.html` | 博客详情页 | common.css, common.js |

## 详细实施步骤

### Phase 1: 提取公共样式和脚本

**文件**: `assets/css/common.css`
- 提取 CSS 变量（颜色、字体、圆角、阴影等）
- 提取 Reset & Base 样式
- 提取导航、按钮、卡片、页脚等公共组件样式
- 提取响应式媒体查询

**文件**: `assets/js/common.js`
- 提取导航滚动效果
- 提取滚动渐入动画 (reveal)
- 提取移动端菜单功能
- 提取工具函数

### Phase 2: 更新首页 (index.html)

1. 引入 common.css 和 common.js
2. 添加跳转到二级页面的链接：
   - 作品集区块添加"查看全部项目"按钮 → projects.html
   - 关于我区块添加"了解更多"按钮 → about.html
   - 服务区块添加"查看服务详情"按钮 → services.html
3. 将项目卡片改为可点击链接

### Phase 3: 创建项目相关页面

**projects.html**:
- 页面标题 + 面包屑导航
- 分类标签筛选（全部/教育/汽车/电商/其他）
- 项目网格展示（响应式 4→3→2→1 列）
- 项目卡片点击跳转到详情页
- 分页或加载更多

**project-detail.html**:
- URL 参数解析 (id, cat)
- 项目主图展示
- 项目信息栏（客户、时间、类型、角色）
- 项目概述和解决方案
- 图片画廊展示
- 相关项目推荐
- 上一篇/下一篇导航

### Phase 4: 创建内容页面

**about.html**:
- 大头像展示 + 个人签名
- 我的故事（设计之路）
- 设计理念阐述
- 详细技能展示（进度条/雷达图）
- 详细时间线（教育+工作+里程碑）
- 兴趣爱好

**services.html**:
- 6项服务的详细展示
- 每项服务包含：详情描述、交付物清单、工作流程
- 整体工作流程展示
- FAQ 常见问题
- 立即咨询 CTA

### Phase 5: 创建博客页面

**blog.html**:
- 文章分类筛选
- 精选文章展示
- 文章网格列表
- 分页导航
- 邮件订阅表单

**blog-post.html**:
- 文章标题、日期、阅读时间
- 文章封面图
- 富文本内容区
- 作者信息
- 相关文章推荐

## 关键设计决策

1. **样式复用策略**: 创建 common.css 共享样式，各页面内嵌专属样式
2. **导航方案**: 二级页面导航链接改为跳转到首页对应锚点或二级页面
3. **项目数据存储**: 使用 JavaScript 对象存储项目数据，通过 URL 参数传递项目 ID
4. **静态网站限制**: 博客内容内嵌在 HTML 中，联系表单使用 Formspree 等第三方服务

## 验证步骤

1. 所有页面样式一致性检查
2. 页面间导航链接测试
3. 响应式布局测试（桌面/平板/手机）
4. 图片懒加载功能验证
5. 项目详情页 URL 参数解析测试
